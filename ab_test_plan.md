# 1. Часть 4. Организация A/B-тестирования

## 1.1 Концепция A/B-теста.

Предложите постановку A/B-теста для сравнения двух версий модели:
- контрольной группы: текущей модели (v1);
- тестовой группы: новой модели (v2).
Описание должно включать:
- способ разделения трафика (например, случайное распределение 50/50);
- продолжительность теста.

### 1.1.1 Постановка A/B-теста.
Метрика - доля корректно выявленных дефолтов.

Гипотеза - Использование алгоритма классификации, основанного на случайных деревьях, увеличит долю корректно выявленных дефолтов на 5 %. 


Аудиторию необходимо разделить на две части:
- контрольную группу: их запросы обрабатываются текущей моделью (v1);
- тестовую группу: их запросы обрабатываются новой моделью (v2).

Аудиторию можно разделить, например, случайным образом 50/50. Но, это будет не совсем корректно, так как случацное распределение не будет учитывать характеристик определенного пользователя и групп пользователей, случайно попавших в ту или иную группу. Это снизит точность интерпретации тестирования. Корректнее будет разделить пользователей таким образом, чтобы каждая из групп отражала характеристики общей выборки. 

Длительность теста: чтобы быть уверенным в результатах тестирования, необходимо проводить тест на достаточном для этого отрезке времени. Длина данного временного отрезка вычисляется с помощью статистических методов, и для этого есть специальные калькуляторы.

Для расчёта размера выборки и длительности проведения теста обычно используют специальные калькуляторы. В интернете можно найти много подобных калькуляторов. 

- Калькулятор Эвана Миллера
Поможет определить размер выборки для задачи повышения конверсии в группах.

- Калькулятор VWO
Позволит рассчитать длительность теста для получения статистически значимого результата. Калькулятор используется для задач повышения конверсии.

## 1.2 Метрики и анализ:

Определите бизнес-метрики для оценки (минимум две):
- основную метрику: F1-score для класса дефолта;
- дополнительную метрику: Precision или Recall (с обоснованием выбора).
Опишите метод статистического анализа:
- какой статистический тест будет использоваться (t-test, z-test);
- как будут строиться доверительные интервалы;
- каков критерий успешности теста.

### 1.2.1 Бизнес-метрики для оценки моделей
1. Основная метрика — F1-score для класса дефолта

Основной метрикой качества выбирается F1-score для класса дефолта.

F1-score является гармоническим средним между Precision и Recall и позволяет одновременно учитывать:

- способность модели находить клиентов с дефолтом;
- количество ложных срабатываний.

Формула метрики:

`
F1 = 2 · (Precision · Recall) / (Precision + Recall)
`​

Использование F1-score особенно важно для задачи кредитного скоринга, поскольку классы обычно несбалансированы:

- клиентов без дефолта значительно больше;
- ошибки обоих типов имеют высокую стоимость для бизнеса.

Высокий F1-score означает, что модель:

- эффективно обнаруживает дефолтных клиентов;
- не создаёт чрезмерное количество ошибочных отказов.

2. Дополнительная метрика — Recall

Дополнительной метрикой выбирается Recall для класса дефолта.

Recall показывает, какую долю реальных дефолтов модель смогла обнаружить:

`
Recall = TP / (TP + FN)
`

Где:

TP — верно предсказанные дефолты;
FN — пропущенные дефолты.
Обоснование выбора Recall

Для банковской системы особенно критична ошибка пропуска дефолтного клиента:

- банк может выдать кредит ненадёжному заёмщику;
- это приводит к финансовым потерям и росту кредитных рисков.

Поэтому важно минимизировать количество ложноотрицательных предсказаний (FN), а Recall напрямую отражает способность модели находить проблемных клиентов.

### 1.2.2 Метод статистического анализа
1. Используемый статистический тест

Для сравнения качества моделей v1 и v2 используется z-test для сравнения пропорций.

Данный тест подходит, поскольку:

- у нас большое колическтво наблюдений;
- результаты классификации можно представить как бинарный исход:
    - корректное определение дефолта;
    - некорректное определение.

При достаточно большом объёме выборки распределение метрик приближается к нормальному, поэтому использование z-test является корректным.


2. Построение доверительных интервалов
Доля корректно выявленных дефолтов - по сути пропорция успехов:

- успех = модель правильно выявила дефолт;
- неуспех = модель ошиблась.

Поэтому здесь нужно использовать доверительный интервал для пропорции и для разности пропорций.

Обозначим:

- Pv1 — доля корректно выявленных дефолтов у модели v1;
- Pv2 — доля корректно выявленных дефолтов у модели v2.
- Pv2 - Pv1 - разница долей.

Далее рассчитываем доверетильные интервалы для долей, таким образом узнаем в каком интевале с заданным уровнем надежности находится доля корректно выявленных дефолтов для каждой модели.

Далее строим доверительный интервал разницы долей, таким образом мы узнаем есть ли статистически значимая разница в долях. 

3. Далее нужно интерпретировать доверительный интервал разницы пропорций. Таким образом можно судить об успешности теста.

- Если обе границы доверительного интервала являются положительными (больше 0), говорят, что вариант v2 более эффективен по заданной метрике, чем вариант v1 с заданной надёжностью.

- Если обе границы доверительного интервала являются отрицательными (меньше 0), говорят, что вариант v2 хуже по заданной метрике, чем вариант v1 с заданной надёжностью.

- Если левая граница доверительного интервала отрицательная, а правая — положительная, говорят, что вариант v2 равнозначен по заданной метрике с вариантом v1 с заданной надёжностью.
